# SimPit

**Multi-machine flight simulator cockpit controller.**

SimPit lets one operator orchestrate a fleet of computers in a home flight
simulator setup — launch X-Plane on every visual machine with one click,
toggle scenery, block updates, push scripts to slaves, and see live status
across the whole rig.

It has been designed for X-Plane 12 multi-machine networks, but the
architecture is sim-agnostic — anything you can drive with a `.bat` or
`.sh` script can be a SimPit operation.

> ⚠️  **Alpha software.** APIs and on-disk formats may shift before 1.0.
> Run only on a trusted LAN.

## What it does

```
                      ┌───────────────────────┐
                      │   SimPit Control      │  (your primary PC)
                      │   • slaves.json       │
                      │   • batfiles.json     │
                      │   • simpit.key        │
                      └───────────┬───────────┘
                                  │  signed UDP/TCP on the LAN
            ┌─────────────────────┼─────────────────────┐
            ▼                     ▼                     ▼
    ┌──────────────┐      ┌──────────────┐      ┌──────────────┐
    │ slave agent  │      │ slave agent  │      │ slave agent  │
    │ (CENTERLEFT) │      │   (RIGHT)    │      │   (...)      │
    │  cascaded/   │      │  cascaded/   │      │  cascaded/   │
    │  local/      │      │  local/      │      │  local/      │
    └──────────────┘      └──────────────┘      └──────────────┘
```

* **One Control app, many slaves.** Slaves are headless executors. They
  have no GUI, no script registration, no cascade flag. Control is the
  sole authority.
* **Cascading scripts.** Mark a script as cascade in Control and its
  content is pushed to every (or selected) slave on sync.
* **Derived state.** Status comes from polling the slaves, never from a
  saved flag — so external changes (X-Plane crash, manual file rename)
  show up immediately.
* **Signed messages.** Every command is HMAC-SHA256 signed with a shared
  key generated by Control. No key, no commands.
* **Cross-platform.** Slaves run on Windows, Linux, or macOS.

## Install

### Prerequisites

* Python 3.10 or newer (Windows, Linux, or macOS)
* `tkinter` for Control (already part of Python on Windows/macOS; on
  Debian/Ubuntu Linux: `sudo apt install python3-tk`)

### From a release wheel

```bash
pip install simpit
```

### From source

```bash
git clone https://github.com/yourname/simpit
cd simpit
pip install -e .[dev]      # editable install + dev tools
```

This installs two console commands:

| Command          | What it runs              |
|------------------|---------------------------|
| `simpit-control` | The GUI controller        |
| `simpit-slave`   | The headless slave agent  |

## First run

### 1. Start Control on your primary PC

```bash
simpit-control
```

A welcome dialog tells you no key has been generated yet. Click
`🔑 Security` → `Generate New`. The key shows up as a 64-character hex
string. **Keep this window open** — you'll need it to set up each slave.

### 2. Install the slave on each visual PC

```bash
pip install simpit
simpit-slave
```

On first run the slave prompts for the key. Paste it from Control's
window and press Enter. The agent saves it to `simpit.key` and starts
listening on UDP `49100` and TCP `49101`.

To run the slave as a service (recommended), see
[`docs/SLAVE_SERVICE.md`](docs/SLAVE_SERVICE.md).

### 3. Register slaves in Control

In Control, click `+ Slave`, enter:

* **NAME**: human label (`CENTERLEFT`)
* **HOST**: the slave's IP or DNS name
* **UDP PORT** / **TCP PORT**: defaults are fine

Within a few seconds the slave card flips to **ONLINE** (yellow).

### 4. Register your scripts

Click `+ Script`. The dialog asks for:

* **DISPLAY NAME**: what shows up in the UI (`Launch X-Plane`)
* **SCRIPT NAME**: base name without extension (`launch_xplane`).
  The slave appends `.bat` on Windows / `.sh` on POSIX automatically.
* **Cascade to slaves**: check this for things every slave should run.
* **SCRIPT CONTENT**: the actual script body (cascade only).
* **STATE PROBE** *(optional)*: lets Control track the script's effect
  state — see [Probes](#probes).

Click `Save`, then `Sync All` in the header to push cascaded scripts to
every online slave. The `▶` button next to a slave's name runs the
script on that slave.

## Probes

A probe lets Control display what the system *currently* looks like
without re-running anything. Built-in probe types:

| Type              | Params              | Meaning                                |
|-------------------|---------------------|----------------------------------------|
| `path_exists`     | `path`              | True iff path exists                   |
| `folder_exists`   | `path`              | True iff path is a directory           |
| `file_contains`   | `path`, `contains`  | True iff file contains literal string  |
| `process_running` | `name`              | True iff a process by that name runs   |
| `script_exit_code`| —                   | Future: run a script as the probe      |

Probe params support `${ENV_VAR}` substitution from the env block sent
with each STATUS request.

**Example** — toggle button for X-Plane custom scenery:

```json
{
  "type": "folder_exists",
  "params": { "path": "${XPLANE_FOLDER}/Custom Scenery" }
}
```

## Debug fleet

To exercise the UI without real network or slaves:

```bash
simpit-control --debug-fleet
```

This wires Control up to a `MockLinkProvider` populated with two fake
slaves (one online, one offline). Useful for screenshots, demos, and
reproducing UI bugs. Mock slave behaviour is configurable — see
[`simpit_control.mock_slave`](simpit_control/mock_slave.py).

## Security

* Every command is signed with HMAC-SHA256 over a canonical JSON
  envelope; signatures fail closed (silent drop).
* Replay protection: messages with a timestamp >30 s away from the
  receiver's clock are rejected.
* Key file (`simpit.key`) is stored 0600 on POSIX. Windows relies on
  user-profile ACL inheritance.
* Slave path-traversal: the slave NEVER accepts a script path with
  `/`, `\`, or `..` — only base names that resolve inside its
  `cascaded/` or `local/` directories.

For the full threat model and remediation steps, see
[`docs/SECURITY.md`](docs/SECURITY.md).

## Architecture

| Package           | Role                                                       |
|-------------------|------------------------------------------------------------|
| `simpit_common`   | Wire protocol, HMAC, OS abstraction, probe engine         |
| `simpit_slave`    | Headless agent — UDP/TCP server, executor, inspector      |
| `simpit_control`  | Store, slave_link, poller, mock_slave, GUI                |
